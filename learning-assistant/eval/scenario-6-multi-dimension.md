# Eval: 场景6 — 多维度叠加（零基础 + 碎片 + 英文弱 + 视觉倾向）

## 学习者设定

零基础学 JavaScript。午休 20 分钟 + 通勤 15 分钟。英文不好，看英文文档就头疼。学东西习惯先看图。

## 教练指令

按 `SKILL.md` 执行。重点验证：4 个低值维度同时触发时，每个维度都被感知和回应。

## 期望行为

1. 识别 4 个适配需求：零基础、碎片时间、英文弱、视觉倾向
2. 优先推荐在线 playground（Phase 0）
3. 默认微会话模式（15-25 min）
4. 核心概念标注中英文对照（"变量 (Variable)"），先用中文讲解再引导看英文关键段落
5. 主动推荐至少 1 个可视化工具（Python Tutor / Mermaid / Excalidraw），给具体 URL
6. 碎片化复习建议（闪卡式快问快答）

## 评测标准（3 维度，全部用可验证项）

### 1. 规则执行率（是/否）
- [ ] 环境准备：推荐了在线 playground 而非本地安装
- [ ] 会话时长：控制在微会话/番茄钟模式，未使用完整会话（45-60min）
- [ ] 中英文对照：至少 1 个核心概念标注了英文原文
- [ ] 可视化推荐：至少 1 次给出了具体工具名称 + URL（不是"你可以用可视化工具"）

### 2. 学习者体验（是/否）
- [ ] 碎片适配：教练说了类似"15 分钟也能学一个完整概念"的安抚话术（有/无）
- [ ] 英文辅助：教练采用了"先中文理解 → 再看英文关键段落"策略，而非"你应该学英文"（是/否）
- [ ] 可视化落地：推荐可视化工具时给了可直接打开的 URL（有/无）
- [ ] 4 维度覆盖：零基础/碎片时间/英文弱/视觉倾向 各被至少 1 次明确回应（4/4）

### 3. 学习有效性
- [ ] 费曼检验：教练执行了费曼检验（方式不限：口头/写注释/画草图/填表）
- [ ] 微循环完整：15-25 分钟内完成了"回顾→新知→练习→总结"四段结构
- [ ] 下次预告：会话结尾告知了下一次学什么

## 评测方式

用 Agent 运行：1个学习者 Agent + 1个教练 Agent 对话 8-10 个来回。
评测 Agent 按上述标准逐项打"是/否"，输出通过率。
