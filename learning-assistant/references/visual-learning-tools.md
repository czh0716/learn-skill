## 1. 核心原则

- **可视化和文字解释是互补的，不是替代的** — 先画图建立直觉，再用文字精确描述
- **不强制所有学习者使用** — 在需求评估阶段了解学习偏好，视觉倾向强者主动推荐
- **画图不是负担** — 手绘草图 > 精美图表，重点在理清思路而非美观
- **先看图再写代码** — 复杂概念的教学顺序：类比 → 草图/流程图 → 代码实现 → 运行验证

## 2. 代码执行可视化

### 2.1 Python Tutor（pythontutor.com）

**适用场景**: 理解变量、作用域、递归、指针、闭包等运行时概念。

**支持语言**: Python, Java, C, C++, JavaScript, Ruby

**使用方式**:
1. 打开 [pythontutor.com](https://pythontutor.com)
2. 粘贴代码，点击 "Visualize Execution"
3. 逐步执行，观察栈帧（Frames）和对象（Objects）的变化

**教学价值**: 把"程序在内存中如何运行"这个抽象概念变成可视化的帧步进，特别适合理解：
- 变量赋值（变量名→值的箭头）
- 函数调用栈（每次调用产生新帧）
- 闭包（内层函数引用外层帧中的变量）
- 递归（栈的层层叠加和逐层返回）

### 2.2 其他执行可视化工具

| 工具 | 适合 | 特点 |
|------|------|------|
| [JavaScript Visualizer](https://www.jsv9000.app) | JS 异步/事件循环 | 可视化 setTimeout/Promise 的执行顺序 |
| [Regexper](https://regexper.com) | 正则表达式 | 将正则转为铁路图 |
| [Regulex](https://jex.im/regulex) | 正则表达式 | 同上，中文友好 |
| [B+ Tree Visualization](https://www.cs.usfca.edu/~galles/visualization/BPlusTree.html) | 数据结构 | USFCA 系列算法可视化 |

## 3. 架构与流程可视化

### 3.1 Mermaid — 代码即图表

**适用场景**: 数据流、请求流程、组件关系、状态转换。

**使用方式**: 在 GitHub Markdown、Notion、Obsidian 中直接写 Mermaid 语法即可渲染。VS Code 安装 Mermaid 插件后可在编辑器内预览。

**教学指令示例**:
> "在你写的代码旁边，用 Mermaid 画一个简单的流程图——数据从用户输入到最终输出，经过了哪些步骤？不要求语法完美，理清思路最重要。"

### 3.2 Excalidraw — 手绘风格草图

**适用场景**: 架构设计讨论、概念关系梳理、白板式思维碰撞。

**使用方式**: 打开 [excalidraw.com](https://excalidraw.com)，直接拖拽画图，无需注册。

**教学价值**: 手绘风格降低"画图要好看"的心理负担。特别适合：
- 学新概念时边画边想
- 对比两种实现方案的架构差异
- 画出"我想实现的效果"再讨论怎么用代码表达

### 3.3 Draw.io / diagrams.net

**适用场景**: 正式的系统架构图、ER 图、UML。

**使用方式**: [app.diagrams.net](https://app.diagrams.net)，可保存到本地或云端。

## 4. 概念关系可视化

### 4.1 思维导图（Mind Map）

**适用场景**: 组织学习笔记、梳理知识体系、记录概念之间的层级关系。

**推荐工具**: XMind, Obsidian Canvas, Markmap（Markdown → 思维导图）

**使用时机**: 
- 每个里程碑结束后，用思维导图整理该阶段的所有概念
- 对比学习时，左右并排放两个思维导图

### 4.2 概念对比表

**适用场景**: 对比相似但不同的概念。

**格式示例**:
```
         | forEach      | map          | filter       | reduce
---------|-------------|-------------|-------------|----------
返回值    | undefined   | 新数组       | 新数组       | 单个值
用途      | 副作用操作   | 映射转换     | 筛选过滤     | 聚合归约
链式调用  | 不可         | 可          | 可          | 谨慎
```

**引导话术**:
> "学了 4 个数组方法，画个对比表吧。格式随意——手写、Markdown 表格、思维导图都行。"

## 5. 在教学中使用可视化的时机

| 教学阶段 | 可视化工具 | 目标 |
|---------|-----------|------|
| 概念探索（POGIL 探索阶段） | Python Tutor | 先运行观察现象，再抽象理解 |
| 概念精讲 | Mermaid / Excalidraw | 用图辅助讲解数据流/执行流程 |
| 费曼检验 | 手绘草图 | "画出来比说出来更直观" |
| 交错综合 | 概念对比表 | 区分相似概念 |
| 30 秒总结 | 思维导图 | 快速梳理知识结构 |
| 项目设计 | 架构图 / 流程图 | 先设计再实现 |

## 6. 引导话术模板

### 视觉倾向型学习者（需求评估时识别）

> "我发现你提到'看到图更容易理解'——这很正常，很多人都是这样。在我们的学习过程中，我会经常建议你用草图、流程图或可视化工具来辅助理解。你习惯用什么画图工具？如果没有特别的，我推荐 [excalidraw.com](https://excalidraw.com)，手绘风格，零学习成本。"

### 概念卡住时

> "这个概念确实抽象。我们换个方式——不用文字，你试试打开 [excalidraw.com](https://excalidraw.com)，画三个方框分别代表'输入'、'处理'、'输出'，然后用箭头连接它们。你刚才写的代码，数据是怎么流的？"

### 费曼检验的视觉替代

> "不用口头解释，你画个草图给我看——如果要把这个概念教给同事，你会画什么图？"

### 30 秒总结的视觉替代

> "用 3 个方框和 2 根箭头，画出今天学到的核心概念之间的关系。"
