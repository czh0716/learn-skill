## 1. 编辑器内交互

### 1.1 `/explain` — 代码解释（费曼检验工具）

**触发方式**: 选中代码 → 右键 → Copilot → Explain，或在 Chat 中输入 `/explain`

**学习场景**:
- **费曼检验**: 先让用户自己解释一段代码，再用 `/explain` 对比 Copilot 的解释，找差距
- **源码阅读**: 选中开源项目中的代码片段，用 `/explain` 拆解逻辑
- **分级理解**: `/explain like I'm a beginner` vs `/explain in technical detail`——对比不同深度的解释

**教学指令示例**:
> "先别看代码，用你自己的话描述这段代码在做什么。然后选中它，按右键 → Copilot → Explain，对比你的理解和 Copilot 的解释。哪里一致？哪里遗漏了？"

### 1.2 Inline Chat (`Ctrl+I`) — 光标级即时交互

**触发方式**: 光标放在任意位置，按 `Ctrl+I`

**学习场景**:
- **即时提问**: 写代码卡住时，Ctrl+I 问 "这里为什么报 undefined？"
- **即时代码审查**: 写完一段函数，Ctrl+I 问 "这段代码有什么潜在问题？"
- **方案对比**: Ctrl+I 问 "这里用 for 循环和用 reduce 哪个更好？为什么？"
- **重构学习**: 选中函数体，Ctrl+I 输入 "用更函数式的方式重写这个，然后解释你做了什么改动"

**教学指令示例**:
> "实现完这个函数后，Ctrl+I 问 Copilot：'review 这个实现，有没有边界情况没处理？' 看看它发现了什么你没考虑到的。"

### 1.3 Chat Diff 内联渲染 — 学习代码变更

**功能**: Code diffs 在聊天内直接渲染，不需跳转到独立 diff 视图

**学习场景**:
- **重构对比**: 让 Copilot 提出重构建议，直接在聊天中看 before/after 差异
- **多方案对比**: 请求两种不同实现方案，在聊天 diff 中并排对比
- **渐进学习**: 每一步的代码变更都有可视化 diff，理解变化过程

**教学指令示例**:
> "让 Copilot 帮你把这个组件用两种方式重写：一种用 class，一种用 hooks。在聊天 diff 中对比两种写法，你观察到什么区别？"

### 1.4 Quick Chat (`Ctrl+Shift+Alt+L`) — 无中断快速提问

**功能**: 弹出一个临时对话框，问完即消失，不进入主聊天历史

**学习场景**:
- 写代码过程中的微问题："这个 API 的参数顺序是什么？"
- 快速查语法："TypeScript 的 satisfies 关键字怎么用？"
- 不打断主学习流程的零碎问题

### 1.5 `@workspace /explain` — 全项目上下文理解

**功能**: 带整个项目上下文解释代码

**学习场景**:
- **理解开源项目架构**: 打开一个陌生项目，`@workspace /explain 这个项目的整体架构是什么？`
- **理解代码流向**: `@workspace 数据是怎么从 API 请求流到数据库的？`
- **新人上手**: 学习如何快速理解一个大型项目

**教学指令示例**:
> "克隆下来这个开源项目后，用 @workspace /explain 让 Copilot 画出项目的架构图和数据流。这能帮你快速建立全局认知。"

## 2. 终端集成

### 2.1 Terminal Agent — 实时终端交互

**功能** (v1.115+): Agent 可与任何可见终端交互，包括 REPL、运行中的脚本、调试会话

**学习场景**:
- **交互式语言学习**: 打开 Node/Python REPL，Agent 可以在其中运行实验代码
- **测试驱动学习**: 先写测试，Agent 在终端运行测试，根据结果调整实现
- **调试实战**: 让 Agent 附加到调试会话，观察变量和执行流

**教学指令示例**:
> "打开一个 Node REPL 终端。我会给你几个数组方法让你实验。每个方法先在 REPL 里试运行，观察结果，然后给我解释。"

### 2.2 后台终端通知 — 自动感知命令完成

**功能** (v1.115): 后台命令完成或需要输入时自动通知

**学习场景**:
- **长时间构建**: 启动一个构建/测试，继续学习其他内容，完成后自动通知
- **多任务并行**: 一个终端在跑测试，另一个终端在实验新代码

### 2.3 `/new` 脚手架 — 快速项目启动

**功能**: `/new Node.js Express TypeScript` 一键创建项目

**学习场景**:
- **项目驱动学习**: 快速搭建学习项目的初始骨架
- **对比学习**: `/new React app` + `/new Vue app`，对比两种框架的项目结构
- **工具链理解**: 观察生成的项目结构，理解 webpack/vite 等构建工具的配置

## 3. Agent 模式

### 3.1 Agent Mode — 自主多文件实现

**功能**: Agent 自动规划、跨文件修改、运行终端命令、运行测试、根据失败迭代

**学习场景**:
- **观察式学习**: 给 Agent 一个需求，观察它如何分解任务、修改文件、运行测试——这是一种元学习
- **渐进式委托**: 
  - 学习初期：Agent 做你做不到的事（搭建环境、配置工具链）
  - 学习中期：Agent 做你不太熟练的事（写测试、重构），你审查学习
  - 学习后期：你做核心逻辑，Agent 做辅助工作

**教学指令示例**:
> "这次让我自己实现核心逻辑。但你可以帮我生成单元测试吗？我想看看你会测哪些边界情况，这也是学习的一部分。"

### 3.2 Plan Mode — 只分析不执行

**功能**: Agent 分析代码库，输出步骤计划，不修改任何文件

**学习场景**:
- **方案思维训练**: 面对一个需求，先用 Plan Mode 让 Copilot 出方案，然后自己实现
- **对比方案**: 自己先写方案，再对比 Plan Mode 的输出，学习架构思维
- **代码阅读训练**: 选中一段复杂代码，Plan Mode 分析其结构和逻辑

### 3.3 Inline Agent Mode — 光标旁代理

**功能** (v1.116): Agent 模式直接在编辑器光标旁工作，不限于侧栏

**学习场景**:
- **焦点学习**: 在正在编辑的文件上直接与 Agent 协作，不切出编辑器
- **增量学习**: Agent 在光标位置做小范围修改，用户观察每次变更并理解
- **对话式重构**: 逐步对话改进代码，每次改一点，每次都有 diff

## 4. 代码审查与质量

### 4.1 Review and Comment — 智能代码审查

**触发方式**: 右键 → Copilot → Review and Comment

**学习场景**:
- **代码质量心理学**: 写完后让 Copilot Review，学习什么是好代码的标准
- **安全性学习**: Copilot 会检查安全问题，用户从中学习安全编码
- **可维护性学习**: 审查结果中的"可读性"和"可维护性"反馈是学习材料

**教学指令示例**:
> "实现完这个功能后，别急着交。右键 → Copilot → Review and Comment，仔细读每一条反馈。挑一条你不太理解的，我们来深入讨论。"

### 4.2 `/fix` — 自动错误检测与修复

**触发方式**: 遇到编译/运行时错误时，点击错误旁的 Fix 按钮

**学习场景**:
- **错误模式学习**: 不只是让 Copilot 修复，而是分析它为什么这样修复
- **费曼检验变体**: "在 Copilot 修复之前，你先猜一下问题在哪里？"
- **类型系统学习**: TypeScript 类型错误时，分析修复方案如何利用了类型系统

### 4.3 `/tests` — 测试用例生成

**触发方式**: 选中函数/类 → `/tests`

**学习场景**:
- **边界思维训练**: 先自己列出所有测试用例，再让 Copilot 生成，对比差距
- **TDD 学习**: 让 Copilot 先生成测试，然后你来实现通过测试的代码
- **测试策略学习**: 观察 Copilot 如何组织测试（边界情况、正常路径、异常路径）

## 5. 会话管理

### 5.1 Conversation Forking — 会话分叉

**功能**: 从任意历史节点分叉，探索不同方向而不会丢失当前上下文

**学习场景**:
- **多方案探索**: "我们试了用 Redux 做状态管理，现在分叉出去，用 Context API 重新实现同一功能，对比两种方案"
- **独立练习**: 分叉一个干净会话用于独立练习，完成后回到主会话继续

### 5.2 Copilot Memory — 跨会话记忆

**功能**: Agent 在 Copilot coding agent、Copilot CLI、和 code review 之间共享知识

**学习场景**:
- **持续性学习记录**: Copilot 能记住你之前学的内容和犯过的错误
- **渐进式项目**: 一个项目跨越多个会话，Copilot 保持上下文

### 5.3 Agent Debug Logs — 可审查的代理日志

**功能** (v1.116): 每次 Agent 交互的完整事件日志持久化保存

**学习场景**:
- **元学习**: 回顾 Agent 的决策过程，学习它的分解任务方式
- **工具链学习**: 观察 Agent 调用了哪些终端命令，按什么顺序
